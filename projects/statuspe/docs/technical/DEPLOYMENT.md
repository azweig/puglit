# Deployment

## Environments
- **Development**: Local environment for testing and development.
- **Staging**: Pre-production environment for final testing.
- **Production**: Live environment serving public status pages.

## Migrations
- Use a migration tool (e.g., Flyway) to manage database schema changes.

## Deployment Process
- **Fly.io**: Deploy the application using Fly.io for global distribution.
- **Vercel**: Optionally use Vercel for frontend deployment if needed.

## Continuous Integration (CI)
- Use GitHub Actions for CI to automate testing and deployment.

## Observability
- Integrate logging and monitoring tools (e.g., Datadog, Sentry) to track application performance and errors.
- Set up alerts for critical incidents or performance degradation.
