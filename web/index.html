The seam is one typed file

No hidden magic. The interview fills a single domain.config.ts. The deterministic spine reads it; the LLM is bound by it and can't invent tables, routes or modules outside it. That's why the generated code is reliable.

// domain.config.ts — the only thing that changes
const mesa: DomainConfig = {
  identity: {
    name: "Mesa",
    tagline: "AI meal planning",
    brandColor: "#10B981",
    languages: ["es", "en"],
  },
  modules: { payments: true, aiLayer: true },
  monetization: { model: "freemium", … },
  landing: { hero: { headline: "Eat well…" } },
}
